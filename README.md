# 📊 File System Report Generator (TypeScript)

## Домашнє завдання до теми: Структурні патерни — Facade та Adapter

---

## 📌 Опис завдання

У цьому домашньому завданні було реалізовано консольну утиліту для аналізу файлової системи з використанням структурних патернів проєктування **Facade** та **Adapter**.

Застосунок виконує рекурсивний аналіз директорії, збирає статистику про її вміст та генерує звіт у різних форматах.

Основною метою роботи було:

- навчитися приховувати складність системи за допомогою фасадів;
- уніфікувати різні формати виводу через адаптери;
- реалізувати багаторівневу архітектуру фасадів;
- розділити відповідальності між компонентами системи.

---

## 🎯 Мета роботи

Реалізувати консольну утиліту, яка:

- аналізує структуру директорій та файлів;
- підраховує кількість файлів та папок;
- обчислює загальний розмір файлів;
- збирає статистику за розширеннями файлів;
- генерує звіт у форматах:
  - JSON
  - CSV
  - XML
- автоматично зберігає звіт у директорію `reports`.

---

## 🏗 Реалізована функціональність

Система підтримує:

- рекурсивний обхід файлової системи;
- підрахунок кількості файлів;
- підрахунок кількості директорій;
- підрахунок загального розміру файлів;
- формування статистики за розширеннями;
- генерацію звітів у декількох форматах;
- автоматичне створення директорії `reports`;
- автоматичне створення файлів звітів із часовою міткою.

---

## 🧠 Застосовані патерни

### ✔ Adapter (Адаптер)

Патерн Adapter використовується для уніфікації різних форматів експорту звітів.

Реалізовані адаптери:

- `JsonReportAdapter`
- `CsvReportAdapter`
- `XmlReportAdapter`

Усі адаптери реалізують спільний інтерфейс:

```ts
ReportAdapter
```

Завдяки цьому система може працювати з будь-яким форматом однаковим способом.

---

### ✔ Facade (Фасад)

У застосунку реалізовано два рівні фасадів.

#### Низькорівневий фасад — AnalyzerFacade

Відповідає за:

- аналіз директорії;
- передачу результату адаптеру;
- формування готового звіту.

Схема роботи:

```txt
DirectoryAnalyzer
        ↓
DirectoryReport
        ↓
ReportAdapter
        ↓
Готовий звіт
```

---

#### Високорівневий фасад — ReportManager

Керує повним життєвим циклом:

- вибором формату;
- створенням адаптера;
- створенням AnalyzerFacade;
- генерацією звіту;
- створенням директорії reports;
- збереженням файлу;
- обробкою помилок.

Клієнтський код працює лише через:

```ts
new ReportManager(format)
```

## 🚀 Як запустити проєкт

Встановити залежності:

```bash
npm install
```

Запуск у форматі JSON:

```bash
npx ts-node src/main.ts . json
```

Запуск у форматі CSV:

```bash
npx ts-node src/main.ts . csv
```

Запуск у форматі XML:

```bash
npx ts-node src/main.ts . xml
```
